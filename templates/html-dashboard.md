# HTML Dashboard Template

Shared layout guidance for all dashboard-style outputs (expansion, upsell, churn, renewal, feature requests, adoption funnel).

## Structure

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>{DASHBOARD_TITLE} — FunnelStory</title>
  <style>
    :root {
      --primary: #1a1a2e;
      --accent: #6c63ff;
      --accent-light: #f0eeff;
      --success: #10b981;
      --warning: #f59e0b;
      --danger: #ef4444;
      --text: #2d2d2d;
      --muted: #6b7280;
      --bg: #f9fafb;
      --card-bg: #ffffff;
      --border: #e5e7eb;
    }
    /* Use a distinctive font — not just Inter */
    @import url('https://fonts.googleapis.com/css2?family=DM+Sans:wght@400;500;700&display=swap');
    body { font-family: 'DM Sans', system-ui, sans-serif; background: var(--bg); color: var(--text); margin: 0; padding: 0; }
  </style>
</head>
<body>
  <!-- Header bar -->
  <!-- KPI row -->
  <!-- Card grid or table -->
  <!-- Footer -->
</body>
</html>
```

## Sections

### Header bar

Dark (`--primary`) background, full width. Contains:
- Dashboard title (left-aligned, white, 24px bold)
- Subtitle: scope + date range (muted, 14px)
- Optional: CSM name or segment label (right-aligned)

### KPI row

3–4 equal-width boxes below the header. Each box:
- Large number (28px bold, `--accent` or status color)
- Label below (12px, `--muted`)
- Use `display: flex; gap: 16px;` for the row. Cards have `--card-bg` background, `--border` border, 16px padding, border-radius 8px.

### Card grid (for account-level dashboards)

One card per account, stacked vertically (full-width) or 2-column grid on wide screens.

Each card contains:
- **Account name** (bold, linked to FunnelStory if `account_id_encoded` available)
- **Key metrics** on one line (health, ARR, renewal, etc. — dot-separated)
- **Insight / rationale** — 1–2 sentences explaining why this account appears. Must cite specific data.
- **Suggested action** — one concrete next step

### Table variant (for compact dashboards)

Use when the user requests a table or >15 rows. Standard HTML table with:
- Sticky header row
- Alternating row backgrounds (`--bg` / `--card-bg`)
- Status pills (colored spans for Critical / High / Medium / Low or G/Y/R)

### Footer

Full-width, muted. `"Generated by FunnelStory · {Date} · {Scope}"`. No external links or buttons.

## Styling rules

- All CSS in a single `<style>` tag. No external JS.
- Max-width 960px centered. Responsive: stack to single column below 640px.
- Status colors: `--danger` for Critical/Red, `--warning` for High/Yellow, `--success` for Green/Healthy.
- Do NOT expose raw internal scores (prediction_score, activity_score) in customer-facing copy. OK in an internal "Method" footnote.

## Evidence attribution

Tag claims in the output so readers know the source:

| Tag | Meaning |
|-----|---------|
| `[Data]` | Directly from FunnelStory semantic DB fields |
| `[Analysis]` | Claude's synthesis or interpretation of multiple data points |
| `[Trend]` | Derived from comparing current vs prior-period values |
| No tag | Structural copy or formatting |

Use tags sparingly — only on key claims in insight/rationale sections, not on every metric.
